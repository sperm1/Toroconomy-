INFLATION SYSTEM
----------------
- Stored in GuildSettings.inflation (percentage, default 0)
- Set via: !dev set-inflation <rate>  (e.g. 3.5 = 3.5%)
- At 0% inflation: 1 UEC = uecRate local units (e.g. $100 = 1 UEC by default)
- Higher inflation weakens server money: effectiveRate = uecRate * (1 + inflation/100)
  → 10% inflation: $110 local = 1 UEC (takes more server money to buy UEC)
  → 50% inflation: $150 local = 1 UEC
- Both buy and sell use effectiveRate so the system is symmetric
- Shown clearly in !exchange info embed with an inflation warning note

SLASH COMMAND RULES (MANDATORY FOR EVERY NEW PREFIX COMMAND)
-------------------------------------------------------------
Every prefix command MUST have a /slash equivalent. The prefix command is NOT
replaced — both must exist and work. Follow this pattern:

SIMPLE COMMANDS (no user-required args):
  Use buildShim from src/utils/slashShim.js and delegate straight to the prefix execute.

    const { SlashCommandBuilder } = require('discord.js');
    const { buildShim } = require('../utils/slashShim');

    module.exports = {
      data: new SlashCommandBuilder()
        .setName('mycommand')
        .setDescription('Does the thing'),
      async execute(interaction, client) {
        const prefixCmd = client.prefixCommands.get('mycommand');
        if (!prefixCmd) return interaction.reply({ content: '❌ Command not found.', ephemeral: true });
        await prefixCmd.execute(buildShim(interaction), [], client);
      },
    };

COMMANDS WITH STRING/INTEGER ARGS:
  Declare options on the SlashCommandBuilder, extract them, build an args[] array,
  and pass it to the prefix execute.

    data: new SlashCommandBuilder()
      .setName('deposit')
      .setDescription('Deposit cash into your bank')
      .addStringOption(opt =>
        opt.setName('amount').setDescription('Amount (number, "half", or "all")').setRequired(true)
      ),
    async execute(interaction, client) {
      const prefixCmd = client.prefixCommands.get('deposit');
      const amount = interaction.options.getString('amount');
      await prefixCmd.execute(buildShim(interaction), [amount], client);
    },

COMMANDS THAT TAKE A @USER MENTION:
  Use addUserOption, then pass the resolved User as the second arg to buildShim.
  The shim's mentions.users.first() will return that user, matching what the
  prefix command expects.

    data: new SlashCommandBuilder()
      .setName('rob')
      .setDescription("Attempt to rob another user's cash")
      .addUserOption(opt => opt.setName('user').setDescription('User to rob').setRequired(true)),
    async execute(interaction, client) {
      const prefixCmd = client.prefixCommands.get('rob');
      const target = interaction.options.getUser('user');
      await prefixCmd.execute(buildShim(interaction, target), [], client);
    },

COMMANDS WITH SUBCOMMANDS (e.g. !org info, !org deposit):
  Use addSubcommand() on the builder. In execute(), get the subcommand name and
  pass it as args[0], then append any extra values.

    async execute(interaction, client) {
      const sub = interaction.options.getSubcommand();
      const amount = interaction.options.getInteger('amount');
      const args = [sub];
      if (amount !== null) args.push(amount.toString());
      await prefixCmd.execute(buildShim(interaction, target), args, client);
    },

DEV-ONLY COMMANDS:
  Copy the isDev() check from prefixCommands/dev.js into the slash file and
  return an ephemeral error for non-devs BEFORE delegating. Keep DEV_IDS in
  sync with prefixCommands/dev.js.

HOW slashShim WORKS:
  src/utils/slashShim.js exports buildShim(interaction, mentionedUser = null).
  It returns an object shaped like a Discord Message with:
    .author         → interaction.user
    .guild          → interaction.guild
    .channel        → interaction.channel
    .createdTimestamp → interaction.createdTimestamp
    .mentions.users.first() → mentionedUser (or null)
    .reply(payload) → interaction.reply / followUp (handles deferred/double-reply safely)
  The prefix command logic never needs to know it's running from a slash interaction.
